# Story 5.5: Sidebar Polish

## Story Info
- **Epic:** [Epic 5 - Professional UI/UX Polish](./epic-5-professional-ui-polish.md)
- **Points:** 5
- **Priority:** P1 (Navigation Polish)
- **Dependencies:** Story 5.1, Story 5.3

## User Story

**As a** user navigating between boards,
**I want** a polished sidebar that feels native and responsive,
**So that** switching contexts is smooth and the interface feels cohesive.

## Background

Current state:
- Sidebar has basic styling
- Board items are functional but plain
- Active states are minimal
- Collapse animation may be abrupt
- Visual hierarchy is flat

Target state (reference: Linear, Notion):
- Sidebar feels like a native app navigation
- Clear active/hover states with smooth transitions
- Proper visual hierarchy (sections, items)
- Smooth collapse/expand animation
- Refined typography and spacing

## Acceptance Criteria

### AC1: Sidebar Container
- [ ] Distinct background from main content
- [ ] Subtle right border or shadow separator
- [ ] Proper padding: 12px horizontal, 8px vertical
- [ ] Smooth width transition on collapse (200ms)
- [ ] Fixed height, internal scroll when needed

### AC2: Board List Items
- [ ] Item height: 36px (comfortable touch target)
- [ ] Icon + text layout with proper spacing
- [ ] Hover: Subtle background highlight
- [ ] Active: Primary-tinted background, bolder text
- [ ] Rounded corners (8px) on hover/active states
- [ ] Smooth transitions (150ms)

### AC3: Section Headers
- [ ] "Boards" section label: 11px, uppercase, muted
- [ ] Proper spacing above/below sections
- [ ] Optional: Collapsible sections (future)
- [ ] Add board button: Visible, properly aligned

### AC4: Add Board Button
- [ ] Positioned at bottom of board list or as list item
- [ ] Icon + "New board" text
- [ ] Subtle styling, not overly prominent
- [ ] Hover state matches board items
- [ ] Consistent with overall sidebar style

### AC5: Collapse/Expand Behavior
- [ ] Toggle button: Visible in header or sidebar
- [ ] Animation: Smooth width transition
- [ ] Content: Fades or slides appropriately
- [ ] Collapsed state: Just icons or fully hidden
- [ ] Keyboard accessible (shortcut hint)

### AC6: Visual Polish Details
- [ ] Scrollbar: Styled to match theme (thin, subtle)
- [ ] Focus states: Visible for keyboard navigation
- [ ] Empty state: Message when no boards exist
- [ ] Board icons: Consistent size and alignment
- [ ] Typography: Proper weight hierarchy
