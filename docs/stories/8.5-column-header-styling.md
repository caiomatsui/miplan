# Story 8.5: Enhanced Column Header Styling

## Status: Ready for Review

## Description
Add subtle gradient/tint styling to column headers to make them more visually distinct while maintaining readability. The gradient should be based on the column's assigned color.

## Acceptance Criteria
- [ ] Column headers have a subtle color gradient or tint
- [ ] Gradient is based on column's color (if set)
- [ ] Default columns without color have a neutral gradient
- [ ] Text remains readable in both light and dark themes
- [ ] Gradient is subtle, not overwhelming
- [ ] Works well with all existing column colors

## Design Approach

### Subtle Gradient Style
```css
/* Example: Top-to-bottom gradient, 10-15% opacity */
background: linear-gradient(
  to bottom,
  color-mix(in srgb, var(--column-color) 12%, transparent),
  transparent
);
```

### Implementation Options

**Option A: CSS gradient with opacity**
```tsx
style={{
  background: `linear-gradient(to bottom, ${columnColor}15, transparent)`,
}}
```

**Option B: Tailwind with dynamic color**
```tsx
className={cn(
  "bg-gradient-to-b from-current/10 to-transparent",
  // with inline style for color
)}
```

**Option C: Use existing color utilities**
If `getColumnHeaderStyles(color)` exists, enhance it to include gradient.

Changes:
- Direction: horizontal → vertical (top to bottom)
- Opacity: 8% → 15% at top, fading to 5% at bottom
- Creates a more noticeable but still subtle color wash
